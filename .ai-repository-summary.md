# Repository Summary: ConvoContentBuddy

## Project Overview

ConvoContentBuddy is an AI-powered autonomous background listener designed to assist users in real-time during technical interviews or coding sessions. The application transcribes audio, analyzes conversational context, identifies algorithmic problems being discussed, and automatically retrieves optimal solutions with minimal user interaction.

### Mission Statement
To create an intelligent, aerospace-grade resilient assistant that helps developers during technical interviews by automatically understanding context and providing relevant solutions without manual input.

### Key Features
- **Zero-Interaction UI**: Dashboard updates organically without mouse/keyboard input
- **Live Transcript Feed**: Real-time visualization of recognized speech
- **Hybrid Vector-Graph Search**: Sub-second algorithmic matching
- **Triple Modular Redundancy**: Aerospace-grade resilience with N+2 failover
- **Multi-language Solutions**: Automatic retrieval of Python, Java, and C++ implementations

## Technology Stack

### Core Framework
- **Language**: C# .NET 10.0 (C# 14)
- **Runtime**: .NET 10.0
- **UI Framework**: Blazor WebAssembly 10
- **Backend**: ASP.NET Core 10
- **Orchestration**: .NET Aspire 10

### AI/LLM Components
- **AI Runtime**: Microsoft.SemanticKernel
- **AI Extensions**: Microsoft.Extensions.AI
- **Primary Model**: Gemini 2.5 Flash
- **Embedding Model**: Gemini text-embedding-004
- **Search Enhancement**: Google Search Grounding

### Architecture
- **Pattern**: 4-layer microservices with Triple Modular Redundancy
- **Service Communication**: SignalR with Redis backplane
- **Search Strategy**: Hybrid vector-graph approach

### Data Storage
- **Vector Database**: Qdrant with gRPC client
- **Graph Database**: PostgreSQL with pgvector extension
- **Cache/Session**: Redis for SignalR backplane

### Infrastructure
- **Containerization**: Docker/Podman
- **Local Development**: Docker Compose via Aspire
- **Logging/Observability**: OpenTelemetry, Serilog
- **Resilience**: Polly with circuit breakers and retries

## Key Architectural Decisions

### 4-Layer Architecture
1. **Audio Input & Transcription Layer**: Browser-based Web Speech API
2. **Context & Intent Analysis Layer**: Semantic Kernel with Gemini 2.5 Flash
3. **Resource Retrieval Layer**: Hybrid vector-graph search
4. **Ambient UI Layer**: Blazor WASM with real-time updates

### Triple Modular Redundancy (TMR)
- Three instances of API.Brain running simultaneously
- Automatic failover and health monitoring
- Redis backplane for state synchronization
- Aerospace-grade 99.9% availability target

### Hybrid Search Chain
- **Vector Search**: Qdrant for semantic similarity
- **Graph Expansion**: PostgreSQL for relationship traversal
- **LLM Verification**: Gemini for result validation
- **Search Grounding**: Google Search for enhanced accuracy
